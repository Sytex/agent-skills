# Add Status Flow to an Object

This skill guides you through adding status flow support to an existing Django model. Status flows allow objects to progress through different statuses with defined transitions.

## Overview

Adding status flow to a model involves changes in both **Backend** and **Frontend**:

### Backend Changes
1. Update the model to inherit from `SytexApprovalFlowProcessModel`
2. Create migrations for the new fields
3. Create a data migration to set up default status flows
4. Update the serializer
5. Register the content type in shared serializers

### Frontend Changes (if applicable)
1. Add status flow filter to search filters
2. Update repository interfaces and mappers
3. Add status flow selection to UI components
4. Update state managers

---

## Backend Implementation

### Step 1: Update the Model

Change the model to inherit from `SytexApprovalFlowProcessModel` instead of `models.SytexModel`:

```python
# Before
from django.db import models as django_models
from shared import models

class YourModel(models.SytexModel):
    status = models.ForeignKey(
        "shared.Status", verbose_name=_("status"), on_delete=models.PROTECT
    )
    # ... other fields

# After
from shared.models import SytexApprovalFlowProcessModel

class YourModel(SytexApprovalFlowProcessModel):
    # Remove the status field - it's now inherited from the mixin
    # ... other fields

    # If you need status_histories GenericRelation with custom related_query_name:
    status_histories = GenericRelation(
        "shared.StatusHistory",
        verbose_name=_("status histories"),
        related_query_name="your_model",  # for backward compatibility
    )
```

The `SytexApprovalFlowProcessModel` mixin provides these fields automatically:
- `status` - ForeignKey to Status
- `status_flow` - ForeignKey to StatusFlow
- `status_flow_version` - ForeignKey to StatusFlowVersion
- `status_step` - ForeignKey to StatusStep
- `current_transition` - ForeignKey to StatusTransition
- `is_rejected` - BooleanField
- `status_histories` - GenericRelation
- `approval_flow_process` - GenericRelation

### Step 3: Create Data Migration for Default Status Flow

Create a data migration to:
1. Create a default StatusFlow for the model
2. Create StatusSteps from existing Status records
3. Create StatusTransitions from existing StatusWorkflow records
4. Update existing objects to use the new status flow

Reference: `app/projects/migrations/0167_create_project_default_status_flow.py`

Key points:
- Use `ContentType.objects.get_for_model(YourModel)` to get the content type
- Filter Status objects by content_type to get statuses for your model
- Use `StatusWorkflow` to build transitions between statuses
- Handle `outside_flow=True` statuses as "from any" transitions
- Create the migration per organization
- Update existing objects to have `status_flow`, `status_flow_version`, and `status_step`

### Step 4: Update the Serializer

Add status flow fields to the serializer:

```python
from shared.models import StatusFlow, StatusFlowVersion
from shared.serializers import StatusStepSerializer, StatusTransitionSerializer

class YourModelSerializer(serializers.SytexModelSerializer):
    status_flow_version = serializers.PrimaryKeyRelatedField(
        required=False, allow_null=True, queryset=StatusFlowVersion.objects.all()
    )

    class Meta:
        model = YourModel
        # ... existing meta

    def get_extra_kwargs(self):
        # ... existing kwargs

        field_grouping["status_flow"] = {
            "group": _("more info"),
            "subgroup": _("configuration"),
            "search_show": False,
        }
        field_grouping["status_step_display"] = {
            "group": "top",
            "subgroup": "status_flow",
        }

        # ... existing field grouping

        extra_kwargs = {
            # ... existing extra kwargs

            "status_flow": {
                "serializer": serializers.ur_serializer(
                    StatusFlow, ["name", "id", "version", "active_version"]
                ),
                "queryset": StatusFlow.objects.all(),
                "required": False,
                "style": {
                    "can_open": True,
                    "ng_readonly": "winCtrl.frontObj.status_flow && winCtrl.frontObj.id",
                    "data_options": "statusflow-list",
                    "options_filter": "{object_type: %s, status: 123}"
                    % ContentType.objects.get_for_model(YourModel).id,
                },
            },
            "status_step_display": {
                "source": "status_step",
                "required": False,
                "read_only": True,
                "serializer": StatusStepSerializer,
            },
            "status_step": {
                "required": False,
                "serializer": StatusStepSerializer,
            },
            "current_transition": {
                "serializer": StatusTransitionSerializer,
                "required": False,
            },
        }

        # Add new fields to the fields list
        fields.append("is_rejected")
        fields.append("is_under_approval")
        fields.append("status_flow_version")
        fields.append("current_transition")
        fields.append("status_step")

    def get_status_flow_on_create(self):
        """Get the default status flow for creation."""
        status = self.initial_data.get("status", None)
        if isinstance(status, dict):
            status = status.get("id")

        status_flow, status_flow_version = self._get_default_status_flow()
        status_step = status_flow.status_steps.filter(
            status_type=status, version=status_flow_version
        )
        return status_flow, status_step
```

### Step 3: Update State Manager

```typescript
// your-model-quick-add.state-manager.ts
import { SearchStatusFlowsUsecase } from '@core/status-flows/application/usecases';
import { StatusFlowSearch, StatusFlowSearchFilter } from '@core/status-flows/domain';

export interface YourModelQuickAddState {
  // ... existing state
  loadingStatusFlows: boolean;
  statusFlowOptions: StatusFlowSearch[];
}

export class YourModelQuickAddStateManager {
  private readonly _searchStatusFlowsUsecase = inject(SearchStatusFlowsUsecase);

  readonly loadingStatusFlows = computed(() => this._state().loadingStatusFlows);
  readonly statusFlowOptions = computed(() => this._state().statusFlowOptions);

  async searchStatusFlows(term: string): Promise<void> {
    this._state.update(state => ({ ...state, loadingStatusFlows: true }));

    const searchFilter: SearchFilter = {
      filters: [
        {
          field: StatusFlowSearchFilter.q,
          operator: SearchFilterItemOperator.contains,
          value: term
        },
        {
          field: StatusFlowSearchFilter.objectTypeModel,
          operator: SearchFilterItemOperator.equals,
          value: 'yourmodel'  // Use your model name here
        }
      ],
      limit: 100,
      offset: 0
    };

    const columns = [StatusFlowSearchColumnField.name, StatusFlowSearchColumnField.isDefaultStart];
    const result = await this._searchStatusFlowsUsecase.execute(searchFilter, columns);

    if (result.isFailure()) {
      this._state.update(state => ({ ...state, loadingStatusFlows: false, statusFlowOptions: [] }));
      return;
    }

    this._state.update(state => ({
      ...state,
      loadingStatusFlows: false,
      statusFlowOptions: result.getSuccess().items
    }));
  }
}
```

### Step 4: Update UI Component

```typescript
// your-model-quick-add.component.ts
providers: [
  // ... existing providers
  SearchStatusFlowsUsecase,
  SearchFilterCriteriaMapper,
  { provide: StatusFlowsRepository, useClass: ApiStatusFlowsRepository }
]

// Add status flow selection
statusFlowOptions = computed(() => {
  const options = this.stateManager.statusFlowOptions();
  return options.map(e => ({
    id: e.id,
    label: e.name || '',
    isDefault: e.isDefaultStart
  }));
});
selectedStatusFlowOption = signal<SelectOption | undefined>(undefined);

// In ngOnInit or constructor with effect:
effect(() => {
  const statusFlows = this.statusFlowOptions();
  if (statusFlows.length > 0 && !this.selectedStatusFlowOption()) {
    const defaultStatusFlow = statusFlows.find(sf => sf.isDefault) || statusFlows[0];
    untracked(() => {
      this.selectedStatusFlowOption.set(defaultStatusFlow);
    });
  }
});
```
